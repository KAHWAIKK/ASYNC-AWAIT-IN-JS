ASYNC VS SYNC IN JS

Governs how we perform tasks which take some time to complete. Js is synchronous in nature, that is it runs from top to bottom , one line after another.This, however has its own drawbacks, look at a scenerio where we nned to make network request to a database on another server, this could take sometime to complete, this action will block the rest of the code from running making the system inefficient as its not the best, hence the need for asyncronous functionality.

That brings us to the use async function which basically start something now and finish later, allowing the code of blocks that follw it to continure running


We normally pass a callback function inside of the async function as a parameter, this callback function is what runs and finishes the code of block


HTTP NETWORK REQUEST

We make http requests to get data from another server, we make these request to API endpoints, for example we may have a weather api which has API endpoints that we can use to acces data from that weather database, the weather database will then send us the requested data as a response

The response data in sent back in JSON format.
